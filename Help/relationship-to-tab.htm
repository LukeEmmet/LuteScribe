Relationship to TAB

TAB by Wayne Cripps is a mature and powerful command line application that offers a wide set of commands to typeset tabulature. It generates typeset output based on textual input.

However TAB has a number of shortcomings which limit its readability and usability for day to day use. LuteScribe augments the TAB format by presenting it in a way that is easier to understand as a player, yet still offers the full facilities that TAB offers. LuteScribe can be thought of as a user friendly wrapper around TAB.

 

  TAB input format feature or limitation   LuteScribe feature to augment or complement TAB
✘ Content is rotated and upside down, making it harder to understand its relationship to the typeset output ✔ LuteScribe represents the tabulature in a familiar horizontal stave format
✘ Flag entries denoting rhythym elements use plain text, making the content harder to understand ✔ LuteScribe uses a custom font so that numerals 1-9 in the flag row appear as flag elements using a familiar tabulature symbol
✘ Dotted rhythms, extended flag commands, fingerings and ornaments result in other stave elements not aligning with each other ✔ LuteScribe's vertical chord format means the main stave elements always correctly line up
✔ TAB offers a wide range of formatting commands ✔ LuteScribe supports all TAB commands with the exception of combined tabulature and standard notation
✘ TAB can only process plain text TAB files ✔ LuteScribe can open TAB, LSML and and other TAB formats such as Fronimo
✘ TAB generates PostScript files which require a specialist application to read or print ✔ LuteScribe creates PDF files which is a very widely supported format for printed output
✘ TAB provides no specific editor tailored to tabulature ✔ LuteScribe provides a user friendly editor that is designed for editing tabulature
