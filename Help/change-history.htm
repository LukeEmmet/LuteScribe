Change history

1.2.1

October 2019

Usability improvements and bug fixes. Improved audio playback. Includes the latest version of TAB.

  • Incorporate latest TAB build 4.3.98
  • Improve TAB file parser to allow comma (,) as note ornament
  • Title field now directly editable
  • Richer feedback when there are warnings or errors from TAB, displayed less obtrusively
  • Playback tools moved back into main window
  • Minor UI tweaks - selected text now shows commands more clearly, and distinguishes the top flag line from the lower lines that denote the courses
  • New playback option - play current stave
  • Fixed reflow bug that it would not always terminate
  • Loading legacy FT2 files - now are reflowed on loading
  • Use FluidSynth for midi playback, with Lute and Piano soundfonts
  • Provide some sample files

1.1.0 - development version

February 2019

Usability improvements for editing - now 50% faster to edit notes and chords using fewer keystrokes. Editing behaviour is now more like a spreadsheet so will feel more predictable and natural.

Added audio playback options.

Summary of changes:

  • Audio playback using WildMidi, FreePats and Naudio
  • Integrated menu of useful and frequently used headers
  • Added default instrument option for playback
  • Added default tempo option for playback
  • Minor bug fixes and usability improvements
  • Editing is more like a spreadsheet, requiring half as many keyboard strokes for each note. Just press the letter then use the cursor keys to move to next cell (up/down/left/right):
    • Cursor keys commit current edit
    • Pressing Del when selecting a cell now clears the current cell
    • Pressing Del when selecing more than one chord clears the content
    • Can also backspace to clear the current cell
    • Keyboard shortcut to remove selected chords like a spreadsheet (Ctrl+-)
  • Insert bar or blank item before current
  • Any pending note edits are committed before previewing the content
  • Visual tweak to make currently edited cell more prominent
  • New Command: Stave->DeleteStave
  • New Command: Edit->SelectAll
  • New Command: Edit->Cut
  • Command line parameter --load-previous to load the last opened file

0.3.3 Beta

December 2018

  • FT3 loading - better detection of fretted and open diapason strings
  • No longer need to type "x" to repeat the last flag. Simply leave the flag empty as you would see in the tabulature
  • Rudimentary undo implemented - can now revert to previous snapshot taken whenever you run a menu command, paste or delete content
  • No longer requires two new lines between headers and content when loading TAB files
  • User option to set preferred default charset and flag style when loading TAB files without any explicitly set
  • Title and author now both set when loading FT3 or JTXML files

0.3.1 Beta

  • Better styling of first bar in each stave
  • New option to control whether the last stave is padded at the end with a flourish (otherwise can result in unattractive stretching of sparse content by TAB)
  • Default charset and flag style is thin robinson style flags and elements, unless explicitly set
  • PDF generation indicates piece number for multi-piece files
  • Improvements to JTXML parser
  • Added recent file list for convenience
  • Better handling of empty elements when generating TAB output
  • Minor change to LSML serialisation format (older LSML files still open), official LSML version 1.0
  • Can open multi-piece TAB files delimited with "%=== new section ==="
  • Section menu to navigate through pieces within same file (TAB, JTXML)
  • Added unit tests
